# AI 面试备战教练」— 精细化产品方案
## 一、产品一句话定义
帮你管理从投递到 Offer 全流程的求职信息中枢 + AI 备战教练。

不是题库，不是面试模拟——而是围绕你真实的求职活动，让每一步都准备充分、不留盲区。

## 二、核心设计原则
原则	含义
围绕真实操作，不编造场景	每个功能对应求职中真实存在的一刻
数据驱动，越用越准	AI 建议基于你积累的历史数据，不是空泛的套话
轻量起步，可渐进扩展	MVP 只做 3 个功能，后续按需叠加
闭环设计	每个操作都有明确的输入和输出，不留"半成品"
## 三、核心功能模块（6 个）
### 模块 1：投递看板
问题：同时投多家公司，进度混乱，容易忘记跟进或错过时间。

功能：

看板视图：3 列 已投递 → 面试中 → 已结束（通过/拒/待选）
每张卡片记录：公司名、岗位名、投递日期、当前进度、下次日期
支持编辑卡片、拖拽调整进度列
过期提醒："公司B 的二面是明天，你准备好了吗？"
输入：手动添加，或粘贴 JD 智能解析出公司名和岗位
输出：一个清晰的全局视图，知道每家公司在哪个阶段、下一步是什么

不做的事：自动爬取投递状态（太复杂、不可靠），纯手动/半自动更新

### 模块 2：岗位匹配分析
问题：投前不知道自己的简历和岗位要求匹配多少、差距在哪、该改什么。

功能：

输入：粘贴岗位描述 + 上传/粘贴简历
输出：
匹配度百分比（基于技能、经验、学历等维度的综合判断）
✅ 你的优势（简历中匹配 JD 的地方）
⚠️ 你的差距（JD 要求但你没有或没体现的）
💡 投递建议（简历中该补充什么、投递前该准备什么）
实现方式：调用 LLM，用结构化 Prompt 对比两段文本，返回格式化的分析结果。

不做的事：不自动帮你改简历（那是另一个产品），只告诉你改什么

### 模块 3：面试复盘
问题：面完后记不清被问了什么、答得怎样，同样的错误下次还会犯。

功能：

面回来花 3-5 分钟用文字描述发生了什么
AI 自动生成结构化记录：
被问到的问题列表（含你答了什么、AI 评估回答质量 1-10 分）
每个问题的改进建议
面试官的"弦外之音"（追问暗示了什么、可能关注什么）
下一轮的预测
输入：用户的一段文字描述（可语音转文字）
输出：一份 Markdown 格式的结构化面试报告，存储在该公司卡片下

关键约束：

AI 不替你"回忆"你没说过的内容——只基于你提供的信息补充分析和判断
每个问题评分需附带理由，不能只给分数
### 模块 4：备战计划
问题：知道下一面要准备，但不知道怎么分配时间、先准备什么。

功能：

输入：距离面试天数 + 上一面的复盘记录 + JD + 岗位方向
输出：按天拆解的备考计划，每天的待办事项
优先复习上一面的薄弱点
按知识点分类安排
每天的任务量合理（不超过 3-4 小时）
支持打卡标记完成
生成逻辑：

1. 从复盘记录提取薄弱知识点
2. 从 JD 提取关键技术方向
3. 按天数分配：薄弱点 > JD核心技能 > 补充知识
4. 每天的任务 = 学习资料 + 练习题目 + 口述演练
不做的事：不自动生成学习内容（那是搜索引擎的事），只告诉你学什么、怎么安排时间

### 模块 5：面试链管理
问题：同一家公司多轮面试之间信息割裂，不知道上一轮暴露的问题下一轮会不会再问。

功能：

一家公司的所有面试记录自动串联成"面试链"
时间线视图：一面 → 二面 → ... → Offer/拒
知识点的"追踪标签"：如果一面在某个问题（如 React diff）答得差，系统标记为"待确认薄弱点"，会在二面备战计划中置顶
面试官信息关联：如果知道二面是谁面，可以参考一面的风格做预判
输入：每次面试复盘后自动构建
输出：一个公司维度的完整时间线和知识图谱

### 模块 6：Offer 比较
问题：拿到多个 Offer 时，纠结于薪资、地点、技术栈，没有结构化的决策框架。

功能：

输入：每个 Offer 的关键信息（薪资、地点、技术栈、晋升、团队、风险等）
输出：
对比表格
按用户自定义权重的综合得分
AI 分析和建议
实现方式：

用户填写结构化表单（不是让 AI 猜）
AI 仅做：维度解释、差异高亮、基于用户偏好给出建议

数据流清晰：

模块 1 的状态变化 → 触发模块 3/4
模块 3 的复盘结果 → 驱动模块 4 的计划生成
模块 3 + 模块 2 的数据 → 构建模块 5 的面试链
模块 5 中的"待确认薄弱点" → 回流到模块 4 的下一轮备战计划

选择	理由

Prisma ORM	TypeScript 类型安全的数据层
本地 JSON (MVP 替代)	前两周不用接数据库，直接存文件，验证后再切

## 八、不做的事（明确边界）
不做什么	为什么
自动爬取投递状态	各家平台接口不同，维护成本无限高
面试模拟/角色扮演	那是另一个产品，先做好管理复盘再说
自动生成简历	太复杂，而且不是核心需求
社区/排行榜	MVP 阶段纯属干扰
语音对话面试	技术复杂度高，后续再加
即时通知推送	MVP 不需要
## 九、总结
这个项目的内核是"信息管理 + AI 分析辅助"，不是什么高深的技术——它的价值在于把你的求职碎片信息组织成结构化的决策支持。

投递前 → 岗位匹配分析帮你判断值不值得投
投递后 → 看板帮你管理进度
面试后 → 复盘帮你知道自己答得怎样、该补什么
备战时 → 计划帮你把时间花在刀刃上
同一家多轮 → 面试链帮你衔接历史信息
最终 → Offer 比较帮你做理性决策
每一步都不复杂，但合在一起就是你的完整求职操作系统。
