## Phase 3: 全链路质量优化

### P0: 批量评分 + 评分锚点
- **改动**: `score_answers` → `batch_score_answers`
- **效果**: API 调用从 N 次降低到 1 次 (省 80%)
- **评分标准**: 1-3/4-6/7-8/9-10 四级锚点，评分一致性提升

### P0: 洞察质量升级
- **改动**: REVIEW_SYSTEM_PROMPT 增加趋势感知指令
- **效果**: AI 能判断薄弱点是改善/退步/稳定，给出具体行动优先级
- **上下文增强**: ContextBuilder 注入薄弱点趋势（↑↓→ 图标 + 分数 + 次数）

### P1: 增量更新 + 趋势算法
- **改动**: `rebuild_profile` → `update_profile_incremental`
- **趋势算法**: 对比最近 2 次 vs 历史平均
  - improving: 最近平均 > 历史平均 + 1.0
  - declining: 最近平均 < 历史平均 - 1.0
  - stable: 变化在 ±1.0 以内
  - new: 只有 1 次记录

### P2: 智能备战计划
- **改动**: PREP_USER_PROMPT 增加难度递进规则
- **难度递进**: Day 1-2 概念 → Day 3-4 实战 → Day 5+ 模拟
- **时间预估**: 每个任务附带 `(45 min)` 标注，前端解析显示 Clock 图标
- **每日总计**: 显示 `total_minutes` 字段
