# Phase 2 — 核心闭环打通 + 上下文注入

> 日期: 2026-04-03
> 目标: 让 投递→复盘→备战→下一轮 真正跑通，AI 建议有实际价值

---

## 一、核心数据流

```
添加公司(带JD)
    │
    ├──AI──→ 匹配分析报告 → 用户决定是否投递
    │
    ▼
添加面试记录
    │
    ▼
面完写复盘 ──AI──→ 结构化报告(含薄弱点/优势/预测)
    │                    │
    │                    ├── 自动保存到 Interview.ai_analysis
    │                    └── 更新 UserProfile 薄弱点统计
    │
    ▼
AI 根据薄弱点 + 画像自动生成备战计划
    │                    │
    │                    └── 保存到 PrepPlan 表
    │
    ▼
下一轮面试时，系统自动注入历史薄弱点
    │
    ▼
复盘对比：上次薄弱点这次答好了吗？
    │
    └── 趋势分析: ↑↓→ 可视化展示
```

## 二、具体任务清单

### Task 1: 用户画像预计算 (UserProfile)
- [ ] 新增 `UserProfile` 数据模型 (SQLite 单表，单用户)
- [ ] 新增 `profile_service.py`: 从历史数据聚合计算画像
  - skills: 从所有 JD 提取高频技能
  - weak_points: 从所有复盘报告聚合，含 count/avg_score/trend
  - strong_points: 从所有复盘报告聚合
  - interview_count / offer_count
- [ ] 写时更新: 每次复盘完成后自动更新画像
- [ ] 读时缓存: 提供 `get_profile()` 方法返回紧凑摘要

### Task 2: 上下文注入引擎 (ContextBuilder)
- [ ] 新增 `context_builder.py`: 按需组装 prompt 上下文
  - `build_review_context(company_id)`: 画像摘要 + 该公司历史 + 最近 2 轮薄弱点
  - `build_prep_context(company_id)`: 画像摘要 + 薄弱点排序
  - `build_match_context()`: 画像摘要 + 职业方向
- [ ] Token 预算控制: 每个场景 < 800 tokens
- [ ] 集成到三个 LangGraph 工作流

### Task 3: 复盘结果自动保存
- [ ] `review_service.py`: 分析完成后写回 `Interview.ai_analysis`
- [ ] 前端复盘页: 分析完成后自动 POST 保存到后端
- [ ] 触发 UserProfile 更新

### Task 4: 备战计划持久化 + 自动填充
- [ ] `prep_service.py`: 生成计划后保存到 `PrepPlan` 表
- [ ] 前端备战页: 打开时自动加载该公司最新复盘的薄弱点
- [ ] 前端备战页: 打卡状态保存到后端

### Task 5: 面试链 API + 薄弱点追踪
- [ ] 新增端点 `GET /api/v1/companies/{id}/interview-chain`
  - 返回: 所有面试按轮次排序 + 薄弱点跨轮次追踪标签
  - 每个薄弱点标记: 首次出现轮次 / 出现次数 / 趋势
- [ ] 前端公司详情页: 渲染面试链时间线 + 薄弱点追踪

### Task 6: 看板增强
- [ ] 首页统计卡片: 投递总数 / 面试中 / 已完成 / 薄弱点 TOP3
- [ ] 公司卡片: 显示匹配度 badge (如果分析过)
- [ ] 过期提醒: 下次事件日期 < 今天时显示红色警告
